Week 3 Ponder and Prove: Using Classes

Overview

Task: Implement the Customer and Account classes shown in the provided UML diagram

Purpose: Learn to create C++ classes

Instructions

Implement the classes shown in the diagram (Customer and Account). First, create a header file to contain your code for each class declaration. Then, create a cpp file for each class definition. You should end up with four files: customer.h, customer.cpp, account.h, and account.cpp.

Do not use pass-by-value for the Customer object in the Account constructor. We don't want to create a copy of a Customer. Instead, we want to pass a reference to the Customer, so we need to use the C++ pass-by-reference mechanism. The private member variable "owner" in the Account class should be a Customer reference rather than a Customer object.

The get_name() member function of the Customer class should return a string composed of the given name and family name concatenated together with a space in between. For example, if the given name is "John" and the family name is "Smith", the get_name() function should return the string "John Smith".

You should include the == operator in each class declaration and implementation.

After you have created the Customer and Account class definitions, create a file called main.cpp and write a simple driver program in the main() function to test your classes. Create an instance of the Account class and test each public method. You will need to create an instance of the Customer class before creating an Account. You should also test each public method of the Customer class.

Thinking about Security

Now that your Customer and Account classes have been implemented, take a moment to think about what could go wrong. Consider these situations:

  • What will happen if you pass a negative number to the deposit method?
  • What if you pass a negative number to the withdraw method?
  • What if you withdraw an amount greater than the balance of the account?
  • What if you deposit an amount greater than can be held by a double?

Consider the implications of these questions. Then go back to your driver program and try each of these situations. Is the result what you expected? What do you need to add to your code to address these questions? Can you think of any other potential bugs or security vulnerabilities with your code?

Review

After completing the instructions:

  1. Review one of the many possible solutions for this task,
  2. Compare and contrast your code with the possible solution's code, and
  3. Make note of any improvements you could make for future tasks.
