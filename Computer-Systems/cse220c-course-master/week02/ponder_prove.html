CSE 220c: Week 02

Ponder and Prove

Week 2 Ponder and Prove: First C++ Program

Overview

Task: Write a console-based calculator program that works with a series of integers.

Purpose: Learn to write C++ programs.

Instructions

You will create a console-based calculator program that will accept a series of integers entered by the user, then display the sum, average, min, and max. The program will first prompt the user for the number of integers they will be entering. Then prompt the user for each number, one at a time. The program will then calculate the sum, average, min, and max for the given numbers and display the result on the console. See an example run of the program below (user input is found after the colon). 


How many numbers? 5
Number 1: 1
Number 2: 5
Number 3: 7
Number 4: 8
Number 5: 9

Sum: 30
Average: 6
Minimum: 1
Maximum: 9

Requirements

  • Store the numbers entered by the user in an array of integers.
  • Use unique_ptr and new to declare an array of the required size.
  • You must use reduce (accumulate) functionality to calculate the sum, min, and max
  • Do not use any built-in sum, min, or max functions. You must write code to calculate them yourself.

After your program is working to calculate the sum, min, max, and average using the reduce functionality, now practice using a map function. Use the C++ transform function on the array of numbers to square each number, storing the results back in the array in place of the original numbers. Use a C++ for_each loop to display the resulting list of numbers.

Next, use a filter function (remove_if) to find all numbers in the array of squares that are less than 1000. Use a for_each loop to display these numbers on the console.

Finally, use the remove_if function again to find all odd numbers in the array of numbers less than 1000. This time, use a C++ for loop. Although this type of loop is less preferred and prone to errors, you will encounter them in code written by others, especially in older C++ code. What do you notice when you try to use a standard for loop? We don't know how many odd numbers were found using the remove_if function! You'll need to do some math with numbers_end and numbers.get() to figure that out. Can you see why using a for_each loop is a better fit for this problem? Display the odd numbers in the array using a for loop.

Here is an example run of the program: 


How many numbers? 5
Number 1: 3
Number 2: 7
Number 3: 16
Number 4: 29
Number 5: 37

Sum: 92
Average: 18.4
Minimum: 3
Maximum: 37
Square of each number: 9 49 256 841 1369
Numbers less than 1000: 9 49 256 841
Odd numbers: 9 49 841

Hints

  • Use a standard C++ for loop to prompt for each number and insert it into the array.
  • Include the algorithm library for the transform, remove_if, and for_each functions
  • Include the numeric library for the accumulate function
  • Include the memory library for unique_ptr

Thinking about Security

Now that you've learned to use the built-in map, filter, and reduce functionality preferred in modern C++, let's look at another way you could have written the code to calculate the sum, min, and max using the older C-type for loop. Then we'll see how this type of code could lead to bugs that introduce a security vulnerabilities into our program.

Prompt the user for five additional numbers and store them in a new array. Then write three functions to calculate the sum, min, and max. Each function should use a for counting loop to iterate over each number in the array. Your three functions should implement the following function prototypes. Note that we must pass in the number of items in the array (the first parameter, count) along with a reference to the array itself. Othewise, the function has no way of knowing how many items are in the array.


int calc_sum(int count, const unique_ptr<int[]>& numbers);
int find_min(int count, const unique_ptr<int[]>& numbers);
int find_max(int count, const unique_ptr<int[]>& numbers);

After you have implemented each function, write some driver code in your main function to call each of these three functions to calculate the sum, average, min, and max again.

What happens if you make a mistake and pass a wrong value for count to one of these functions? For example, if your array of numbers has 5 items, what if you accidentally pass the value 6 as the first parameter to your calc_sum function?

What happens if you accidentally write a for loop that counts one number too high so you end up trying to access an item in the numbers array that is outside the bounds of the array?

Review

After completing the instructions:

  1. Review one of the many possible solutions for this task,
  2. Compare and contrast your code with the possible solution's code, and
  3. Make note of any improvements you could make for future tasks.
