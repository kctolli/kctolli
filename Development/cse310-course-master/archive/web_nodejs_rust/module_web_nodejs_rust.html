Discovery Module—Web Sites using NodeJS and Rust

Overview

From the article What Is Rust And Why Is It So Popular?, we read that Rust has been on Stackoverflow's most loved languages list for years and that it solves many pain points of other programming languauges. It is growing quickly and has incredible potential in many different fields.

Likewise, with more than a billion downloads, Node.js is growing at an incredible rate. It thrives in building real-time applications, the Internet of Things, and microservices. Many advantages of using Node.js can be read about here: 6 Reasons Why You Should Use Node.js for Enterprise-Level Organizations.

Considering these two things, this module is meant to build your expertise in both of these areas. In the blog post Rust and Node.js: A Match Made in Heaven, Anshul Goyal explains:

“Node.js is a very popular JavaScript runtime for writing backend applications. Its flexibility and nonblocking nature have made it the premier choice for API consumption.

“Since it is a scripting language, JavaScript can be pretty slow. But thanks to V8 optimization, it is fast enough for practical applications. That said, Node.js is not good for heavy lifting; since it’s single-threaded, it is dangerous to block the main thread for doing long calculations. That’s where worker threads come in. Node.js has support for worker threads, so it can be used to perform long calculations.

“As great as worker threads are, JavaScript is still slow. Moreover, worker threads are not available in all supported LTS versions of Node. Fortunately, we can use Rust to build a native add-on for Node.js. FFI is another alternative, but it’s slower than the add-on approach. Rust is blazing fast and has fearless concurrency. Since Rust has a very small runtime (or “not runtime”), our binary size should also be pretty small.”

Overall, there are many reasons to consider using Rust in a Node.js project. As mentioned above, Rust is able to perform heavy tasks with extreme efficiency and security. While most tasks you could technically perform with Node.js, many will perform much better with Rust because of the CPU-intensive processing they require. Some examples include cryptography, image processing, video processing, or anything that utilizes a lot of data. For example, several large apps that perform complex tasks in programming languages similar to Rust are Amazon, Google and Facebook.

Module Requirements

Implementation

You need to complete the following:

  • Node.JS Application

    • To get started with Node.js, you may want to follow this tutorial: Node.js Tutorial in Visual Studio Code. Alternatively, you can start on the Node.js install page.

    • Download nodejs_template.zip as a base for your web application.

    • Inspect the code and run it. Begin to think about ways you could build an application that would effectively utilize Rust.

    • Your web application must contain at least one page and should effectively utilize Rust.

    • It should do something beneficial and be useful enough that someone would want to use it.

    • You are not required to push this project to the cloud to complete this module. You can develop and complete it locally (localhost) and submit a ZIP file upon completion.

  • Rust Module

    • Incorporate a Rust module into this application.

    • This can be a simple module, but should complement your Node application.

    • To get started with Rust, go to Rust's Getting Started page. Be sure to view the guidance resources below as well.

Documentation

The program needs to be published to a public GitHub repository as part of your personal software portfolio. Be careful not to include any passwords or keys in your published material. Ensure that the repository has a good description and is tagged appropriately. The README.md file in your repository should be well-documented, with at least the following information:

  • A description of your program.

  • How to set up the development environment, including all tools. Provide detailed steps so that you or another engineer could replicate the environment.

  • How to build, execute, and use your program as applicable for your development environment.

  • A list of useful websites that you found to assist your development work.

Stretch Challenges

After your complete the module requirements, you can earn additional bonus points by completing one of the following stretch challenges:

  • Deploy this application to the cloud so that anyone can view it with the URL.

  • Incorporate a database such as Firebase, MySQL, MongoDB, or PostgreSQL to save application data.

  • Incorporate threading into the Rust module.

The implementation of a stretch challenge must be done within the same GitHub repository and reflected in the descriptions found in your README.md file.

You may not be awarded bonus points if the module requirements were not completed satisfactorily.
