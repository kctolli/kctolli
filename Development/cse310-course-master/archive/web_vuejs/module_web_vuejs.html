Discovery Module - Progressive Web Apps with VueJS

Overview

There are many different types of applications: desktop apps, mobile apps, web apps, and many others. Many companies will expend great effort to build a web application, and then expend close to the same level of effort to develop a mobile or desktop application for their customers to use. This is often done because there are many capabilities that native apps have that websites simply cannot have, such as working with file systems, or using device functionalities such as haptic feedback or push notifications.

A progressive web app is built using web technologies, but acts and feels like a native app on a mobile device. For companies or developers needing to develop software for both the web and mobile/desktop apps, progressive web apps can save an incredible amount of time and money in the short and long term. This is because, even after deployment, apps will be easier to maintain and update across the board. Alex Russell, who coined the term progressive web app, said, "Progressive web apps are just websites that took all the right vitamins."

You can read more from sites like the following:

  • Progressive Web Apps 101: the What, Why and How (freecodecamp.org)

  • What Are Progressive Web Apps? (web.dev)

Module Requirements

To complete this module, you need to do the following:

  • Learn how to create a web application using Vue.js

  • Deploy this application as a PWA (progressive web app)

  • The app that you create should do the following:

    • Have a professional look and feel

    • Accomplish something meaningful

    • Be able to be installed on the operating system of your computer (MacOS, Windows or Linux)

    • Be able to be installed on the operating system of your mobile device (Android or iOS)

  • The program needs to be published to a public GitHub repository as part of your personal programming portfolio. Be careful not to include any passwords or keys in your published material. Ensure that the repository has a good description and is tagged appropriately. The README.md file in your repository should be well-documented with at least the following information:

    • A description of your program.

    • How to set up the development environment, including all tools. Provide detailed steps so that you or another engineer could replicate the environment.

    • How to build (if applicable), execute, and use your program. This includes any settings that a user would have to modify to allow the installation of a PWA on their devices.

    • Screenshots of your software running.

    • A list of useful websites that you found to assist your development work.

Stretch Challenges

You can earn additional bonus points by completing one of the following stretch challenges:

  • Incorporate the necessary flags and code to be able to use the camera of a mobile device and view a new picture in the app.

  • Incorporate a data layer such as Firebase to save data from your application.

  • Modify the desktop version of this application to access and modify the file system of the computer it is installed on.

The implementation of a stretch challenge must be done within the same GitHub repository, and reflected in the descriptions found in your README.md file.
