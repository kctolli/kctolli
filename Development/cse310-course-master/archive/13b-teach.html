W13 Teach B: Team Project

Peer Feedback

Each individual will be assigned to review one of the projects. You will receive the document submitted by one of the teams for their project demo. The document contains the link to the public GitHub repository. If the demo was done as a video, the link will also be provided in the document.

Instructions

  1. If a video is provided, watch the video. Otherwise, you should refer to your notes from the live demos you attended during class.

  2. Review the material in GitHub including the README.md file.

  3. Identify one suggestion for improvement to the project.

  4. On the project GitHub site, raise a GitHub issue for your suggestion. Provide a meaningful title and a detailed description. The team will review your suggestion. You should not feel offended if they choose not to implement your suggestion.

  5. Take a screenshot of the GitHub issue that you created.

  6. Download the Peer Feedback template and place the screenshot in the document.

Rubric

This is a pass/fail assignment. If any part of the peer feedback is incomplete or lacking detail, then you will receive a fail.
