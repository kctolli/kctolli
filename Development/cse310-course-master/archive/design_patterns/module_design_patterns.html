Module Requirements

Implementation

Your are strongly encouraged to use the Guidance section below when implementing these requirements. You need to complete the following:

  • Write three separate programs that demonstrate three separate design patterns.

  • You may use any programming language.

  • Each program should perform a useful task. The programs should be interesting and complex enough so that you will be excited to share them with potential employers and co-workers. In general, each program should be at least fifty lines of code.

  • Each program should be of your own creation and not the result of completing an online tutorial. Using tutorials is a good way to learn but you should follow this up with the creation of your work.

  • Ensure you code is well documented with comments.

Documentation

Use the following README.md template to create your documentation for your public GitHub repository. It is important that you complete all sections of the template before submiting your module.

In your README.md file, you will need provide a link to a YouTube video that introduces your softare, demos the execution of the software, and walksthrough the code. Your video should be 3-4 minutes in length.

Stretch Challenges

After you complete the module requirements, you can earn additional bonus points by completing a fourth program that implements an additional design pattern.

The implementation of a stretch challenge must be done within the same GitHub repository, and reflected in the descriptions found in your README.md file.

You may not be awarded bonus points if the module requirements were not completed satisfactorily.

Grading Rubric

This module will be graded as follows, with only five possible numeric grades (there is no scale):

Grade Description
100 (A) Module is mastered, implementing all module requirements correctly with a detailed and professional README.md file in GitHub. A potential employer would likely be interested in discussing this with you if it was relevant to the job opening. You are encouraged to continue to maintain your software to both improve and add new functionality.
89 (B+) Module is well-understood, implementing most module requirements correctly, OR your README.md file may lack detail. A potential employer might be interested in discussing this with you during a job interview if it was relevant to the job opening. You should look for opportunities to improve your software and documentation.
75 (C) Module is partially understood, implementing some module requirements correctly. A potential employer would likely not be interested. You should continue to improve this software before sharing with a potential employer.
60 (D-) Module was attempted but not understood. A potential employer would likely not be interested. You should continue to improve this software before sharing with a potential employer.
0 (F) Module was not submitted.

Up to a 10% bonus can be achieved by completing one of the stretch challenges. Additional bonus points are not given for doing multiple stretch challenges. However, you will improve your personal software portfolio if you continue to set aside time to develop new functionality.
