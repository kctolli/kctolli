Overview

Did you know that there are close to three million applications in the Google Play Store? Did you also know that the average Android developer makes well over $100k/year? Android is everywhere, and the ability to make Android applications has never been more sought after or desirable than it is now. Android applications are typically written in Java or Kotlin using Android Studio, and while many applications are published to the Google Play Store or other application markets, developers are able to create applications and distribute them without such deployments. This module is intended for you to learn how to develop Android applications quickly.

You can read more from sites such as the following:

  • Android Apps - Wikipedia (reference site)

  • Android Studio (official site)

Module Requirements

Implementation

You need to complete the following:

  • Create a native Android application using Android Studio (using Java or Kotlin)

  • This application should do the following:

    • Have at least one screen

    • Be interactive with the user

    • Be helpful in performing a task that you, or someone you know, do on a frequent basis

    • Run successfully on an Android device (or on an emulator)

Documentation

The program needs to be published to a public GitHub repository as part of your personal software portfolio. Be careful not to include any passwords or keys in your published material. Ensure that the repository has a good description and is tagged appropriately. The README.md file in your repository should be well-documented, with at least the following information:

  • A description of your program.

  • How to set up the development environment, including all tools. Provide detailed steps so that you or another engineer could replicate the environment.

  • How to build, execute, and use your program as applicable for your development environment.

  • Screenshots of your software running.

  • A list of useful websites that you found to assist your development work.

Stretch Challenges

After you complete the module requirements, you can earn additional bonus points by completing one of the following stretch challenges:

  • Use something in Android that requires permission (for example: location, camera, microphone, etc.)

  • Incorporate multiple screens or activities

  • Store data either in local storage (files or shared preferences)

The implementation of a stretch challenge must be done within the same GitHub repository, and reflected in the descriptions found in your README.md file.

You may not be awarded bonus points if the module requirements were not completed satisfactorily.

Guidance

You should consider using Android Studio and either Java or Kotlin (depending on which language you know best). The app should run on an Android phone. If you don't have an Android phone, you can use the emulator provided within Android Studio. Here is a list of websites that you might find useful when creating an Android app:

  • Build Your First App

  • Android Basics in Kotlin

The following links may be useful for some of the stretch challenges:

  • Permissions Overview

  • Starting Activities with an Intent

  • Saving data in Shared Preferences
