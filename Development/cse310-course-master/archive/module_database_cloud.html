Discovery Module—Cloud Databases

Overview

Cloud data storage is a convenient solution which removes the requirement for an individual or company to maintain their own database server. Tools like Google Firebase, Amazon DynamoDB, and MongoDB will provide you a database which you can access using an API (function calls from a library). The use of these cloud databases is frequently free for simple projects, but will cost when scaled up. Cloud databases provide a mix of various types of databases, including relational and key/value. A key/value database (which is recommended in the instructions below) will store data into maps that can be queried either in whole, in parts, or in filtered parts. A key/value database is also commonly called a NoSQL database. The APIs for each database will vary and provide these capabilities.

You can read more on sites like the following:

  • Overview of Cloud Databases - MongoDB (company-sponsored site)

  • An Introduction to Cloud Databases: Chapter 1 (electronic book)

  • NoSQL Tutorial - Guru99 (tutorial site)

Module Requirements

Implementation

You need to complete the following:

  • Write a program that interacts with a cloud database using NoSQL. The purpose of the program and the type of data stored is up to you. The program can be written in any programming language and use any cloud NoSQL database. See the notes below for guidance about this step.

  • Create a database for your program to use.

  • The program should demonstrate the following capabilities working with the database:

    • Insert new data into the database.

    • Modify existing data in the database.

    • Delete existing data from the database.

    • Retrieve all data from the database.

Documentation

The program needs to be published to a public GitHub repository as part of your personal software portfolio. Be careful not to include any passwords or keys in your published material. Ensure that the repository has a good description and is tagged appropriately. The README.md file in your repository should be well-documented, with at least the following information:

  • A description of your program.

  • How to set up the development environment, including all tools and online accounts. Provide detailed steps so that you or another engineer could replicate the environment. Remember, you should not include any passwords or keys in the documentation.

  • How to build, execute, and use your program as applicable for your development environment.

  • Screenshots of your software running.

  • Description of the API commands you used in the program to interact with the database.

  • A list of useful websites that you found to assist your development work.

Stretch Challenges

After you complete the module requirements, you can earn additional bonus points by completing one of the following stretch challenges:

  • Receive notifications in your software when data in the cloud changes.

  • Require the user of your software to create an account with your software and log in, and only display database information belonging to that user.

  • Organize data in your database based on the user that is using the software so that only data related to that user is queried and modified.

  • Store pictures in the database.

The implementation of a stretch challenge must be done within the same GitHub repository, and reflected in the descriptions in your README.md file.

You may not be awarded bonus points if the module requirements were not completed satisfactorily.

Guidance

You should consider using Python and Firestore (part of Google Firebase) for this module. Here are some useful notes when using this suggested environment:

  • Use firebase-admin to access a Firebase database from Python. You can use pip to install: pip install firebase-admin

  • You will need to create a free Firebase account and create a Firebase Project.

  • During the process of getting your account and database set up, you will need to download a Service Account File, which is a JSON file. This file along with the project name will be needed when you initialize your connection to the Firestore database within your code. The Service Account File is secret and should not be shared in the GitHub repository.

Here is a list of websites that you might find useful when working in the suggested environment:

  • Firestore Tutorial (official site)

  • Firebase Console (official site)

  • Google Authentication—Creating a Service Account (official site)
