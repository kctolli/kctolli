12 Prepare : Checkpoint A - Non-member Operator Overloading

After completing (or while completing) the preparation material for this week, complete the following exercise.

Overview

This checkpoint is intended to help you practice the syntax of operator overloading with non-member functions. You will work with the money class we have used in past assignments and implement a set of operators.

Instructions

  1. Begin with a working (but simplistic) implementation of a Money class that contains integer values for dollars and cents. You can copy this code (and a driver that file that you'll need) to your directory via the following:
    2. mkdir check12a cd check12a cp /home/cs165new/check12a/* .
  2. This class has the following class diagram:
    3. Money
    -dollars : int
    -cents : int
    +Money()
    +Money(dollars, cents)
    +getDollars() : int
    +getCents() : int
    +setDollars(int) : void
    +setCents(int) : void
    +prompt() : void
    +display() : void
  3. You need to overload the following operators as non-member functions (in other words regular functions outside the class). Please note that the prototypes for these functions should still be in the Money.h file, but should come after the class. Then the bodies of these functions should be in the Money.cpp file.
  4. Implement the following:
    • == (Equivalence) - Check if the dollars and cents amounts are equal.
    • != (Not Equal) - the opposite of the == operator.
    • << (Stream insertion aka the "output" operator) - This should behave like the display function and allow you to output a Money object to the console, with a statement such as "cout << myMoney;" that would produce output in the following format: $3.45 or: $3.05 or: $0.61
  5. Fill in the main function (in check12a.cpp) to use your operators as noted.

Sample Output

The following is an example of output for this program:

Dollars: 10 Cents: 67 Dollars: 11 Cents: 67 The values are: $10.67 and $11.67 Using the == operator, they are: Not Equal Using the != operator, they are: Not Equal

Another example would be:

Dollars: 8 Cents: 17 Dollars: 8 Cents: 17 The values are: $8.17 and $8.17 Using the == operator, they are: Equal Using the != operator, they are: Equal

Testbed

An auto-grading testbed script is provided for you to help evaluate your program. This same testbed script will be used to grade your program. It is pass/fail, so your program must pass the testbed completely for you to receive credit for this assignment. You may run the testbed as many times as you like.

Please note that it is possible to do this assignment without overloading the operators, but this will not help you understand the concept, and you will not receive credit if you do not follow the instructions.
