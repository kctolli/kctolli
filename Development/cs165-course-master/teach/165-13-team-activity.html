13 Teach : Team Activity - The Rule of Three

Overview

This activity will help you practice the syntax of copy constructors, assignment operators, and destructors.

Your assignment is to implement each of these for a Course class that contains a dynamically allocated array of Students.

Instructions

Begin with the code at: /home/cs165new/ta13/*

Notice that because a Course has a dynamically allocated array, it needs a destructor, a copy constructor, and an overloaded assignment operator.

Core Requirements

  1. Implement a constructor that takes an integer for the size. It should set the size member variable and then allocate the array of Students to that size. Then, implement the getStudent and setStudent methods of the Course class that should get and set students at the specified index (e.g., setStudent(3, john) should set john to be at index 3 in the array).
  2. Implement a destructor for the Course class that deallocates the array. Then, display a message, "Cleaning up course: [course-name]". Please note that in practice we would not actually output a message when cleaning up a message, but while practicing this helps us see that it is actually being called.
  3. Implement a copy constructor for the Course class that performs a deep copy (i.e., allocates a new array and copies each student over). Also, overload the assignment operator for the Course class to perform a deep copy.

When you complete these requirements, show your code to your instructor to "pass it off."

Stretch Challenges

After completing the above steps. Make sure that everyone on the entire team is to this point and understands the material. Then, if you have time, move onto the following stretch challenges.

  1. Overload the output operator for both Student and Course. Remove the display functions for each and replace any references to it with calls to your output operator.
  2. Change the Student class to use a dynamically allocated char[] for the name instead of a object of the string class. Leave the parameters and the return type of getName and setName as they were, but change the bodies of these functions to convert between the character array and the string data types.
  3. Because the Student class now has dynamically allocated data, implement a destructor, copy constructor, and assignment operator for it as well. See if you can add something to the driver program to test that your code is working the way you expect.
