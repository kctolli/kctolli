Using AsyncTask

Objectives

  • Create a multithreaded program in Android using AsyncTask.

This assignment builds upon the team activity for this week. Before continuing, please read and complete this week's Team Activity.

Assignment

Your assignment is to:

  1. Finish any parts of the team activity that you did not complete as a team.

  2. Change the app so that the saving and loading of the file happen on a background thread, so that they do not cause the Graphical User Interface (GUI) to stop responding to the user.

    After reading or writing each line, update the progress bar. (i.e. Pressing the Create button should set the progress to 0, and it should steadily fill up as each line is written. Pressing the Load button should again set it to 0, and it should steadily fill up as each line is read.)

    Remember that background threads can't update the GUI directly.

You are welcome to use any multithreading option that Android has to offer, but AsyncTask is particularly suited for this type of scenario. AsyncTask has several methods that run code on the background and main GUI threads at the appropriate times. There are some weird generic parameters to get used to, but when you put everything in place, it should take care of all the threading for you.

When complete, upload your code to I-Learn, then check out the instructor's code and also, don't forget to complete the Ponder reflection for the week.
