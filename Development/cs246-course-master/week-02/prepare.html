Software Design Principle

Program to an interface, not an implementation.

Using interfaces makes our code more flexible. To understand why this is:

  1. First, read this official description of interfaces from Oracle's Java Documentation in order to get a basic idea of what an interface is.

  2. Next, watch this video, which shows how interfaces can help us avoid a particular design issue:

  3. Then, read the first answer to this Stack Overflow question to understand the main advantage of interfaces.

  4. Finally, read the top four answers to this Stack Overflow post to see other advantages and thoughts about interfaces.

After reading that material, you should understand the advantages provided by programming to an interface rather than an implementation.

Namely, it allows our code to move away from this design: "I need this specific class in order to do this work." and instead use this more flexible idea: "I can use any class that can do this kind of thing in order to do this work."
