Version Control with GitHub

Objectives

  • Write Java code and configure it using git and GitHub

  • Practice using branches in Git

Team Development

It this exercise, you will create a simple Java program and put it into GitHub. You will then ask one of your team members to make a change to your project. You will also make a change to another team member's project.

    Step 1: Create a project in IntelliJ called HelloFrom<Name> (if you first name was Bob, then this would be HelloFromBob).

    Step 2: Before writing any code, import your project into Github by going to menu option VCS, selecting "Import into Version Control", and selecting "Share Project on Github".

    Step 3: You will likely be prompted to login to Github. After providing your credentials, you will prompted to create a GitHub repository. Make sure that the repository is public (don't select the private checkbox). You will likely be prompted to commit basic IntelliJ project files (e.g. modules.xml). Select Add to commit those files and push to your repository in GitHub.

    Step 4: Goto GitHub and verify that you see your new repository.

    Step 5: Write code in your IntelliJ project to say "Hello, my name is <your name>". Run your program to make sure it works.

    Step 6: Using either the Git tools in IntelliJ or using the Git Bash command line, commit and push your changes. Goto GitHub and very that you see your changes.

    Step 7: While in Github, click on Settings and goto to Collaborators. Add all members of your team as collaborators for your project.

    Step 8: At least one member of your team must make a modification in your project. You must make at least one modification in another team members project. To make a modification in another project, goto IntelliJ and select (from either the VCS menu or when you first start IntelliJ and no other project is open) Checkout from Version Control (select git). You will need to type in the URL for your teammate's repository in GitHub. You will also specify where you want the project to be placed on your hard drive.

    Step 9: Add a line of code in the program to display "Hello <your team members name>, my name is <your name>." Run the program to make sure it works.

    Step 10: Using either the Git tools in IntelliJ or using the Git Bash command line, commit and push your changes. If a merge conflict happens, you will need to manually fix the merge conflict.

    Step 11: After someone has successfully updated your project and you have successfully updated someone else's project, then you are done and can take the quiz in I-Learn.
