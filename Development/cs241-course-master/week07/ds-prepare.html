Recursion

Overview

To prepare us for a few of the upcoming data structures and algorithms we will study, we need an additional tool in our tool belt: Recursion.

At this point, we are comfortable having functions call other functions. And even having function A call B, then B call C, and even C call D doesn't really scare us very much. The computer doesn't somehow lose track of where it is, of the value of variables in function A, just because other functions are being called.

But what if function A called itself? Is that even legal? Does it just cause an error? This is called recursion.
